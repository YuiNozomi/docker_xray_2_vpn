## 二、国内测试阶段（当前）

### 测试目标
✅ **验证连接建立成功**
✅ **验证局域网内连通性**
⚠️ **暂不验证访问外网**（因为国内服务器本身无法直接访问外网）

### 测试步骤

#### 1. 局域网内测试（同一网络下的设备）

**在另一台电脑（客户端）上：**

1. **配置 v2rayN**：
   - 地址：`192.168.1.203`
   - 端口：`10086`
   - UUID：`9db5d1b8-8463-4e62-8f1f-f065a1b456d3`
   - 传输协议：`ws`
   - 路径：`/ray`
   - Host：`vpn.example.com`
   - TLS：✓ 开启

2. **测试端口连通性**：
   ```powershell
   Test-NetConnection -ComputerName 192.168.1.203 -Port 10086
   ```
   应该显示 `TcpTestSucceeded : True`

3. **测试连接延迟**：
   - 在 v2rayN 中点击"测试延迟"
   - 应该显示具体数值（不是 -1）

4. **测试局域网资源访问**：
   ```powershell
   # 访问服务器上的资源（如果能访问百度等国内网站）
   curl -x socks5://127.0.0.1:1080 https://www.baidu.com
   ```

## 三、国外部署阶段（实际生产环境）

### 部署到国外后

当你的 Windows 服务器部署到国外后：

#### 1. 获取服务器 IP

**在国外服务器上运行：**
```powershell
# 获取公网 IP
curl https://api.ipify.org

# 获取局域网 IP
ipconfig | Select-String "IPv4"
```

#### 2. 配置路由器端口转发（如在局域网内）

如果需要从外网访问：
- 登录路由器管理界面
- 添加端口转发规则：
  - 外部端口：`10086`
  - 内部 IP：`192.168.x.x`（服务器的局域网 IP）
  - 内部端口：`10086`
  - 协议：`TCP`

#### 3. 客户端配置（外地电脑）

**在客户端 v2rayN 中：**

| 字段 | 值 |
|------|-----|
| 地址 | 服务器的**公网 IP** 或域名 |
| 端口 | `10086` |
| UUID | `9db5d1b8-8463-4e62-8f1f-f065a1b456d3` |
| 传输协议 | `ws` |
| 路径 | `/ray` |
| Host | `vpn.example.com` |
| TLS | ✓ 开启 |

#### 4. 测试外网访问

```powershell
# 测试 Google
curl -x socks5://127.0.0.1:1080 https://www.google.com

# 测试 IP 变化
curl -x socks5://127.0.0.1:1080 https://api.ipify.org
# 应该显示服务器的出口 IP
```

## 五、常见问题排查

### 问题 1：客户端连接失败

```powershell
# 检查服务器端口是否监听
netstat -an | findstr 10086

# 检查防火墙规则
Get-NetFirewallRule -DisplayName "Xray VPN Port"

# 检查 Docker 容器状态
docker compose ps
```

### 问题 2：延迟显示 -1

1. 确认 v2rayN 配置正确
2. 确认服务器端口可访问
3. 确认服务器本身能访问外网（国外部署后）

### 问题 3：无法访问外网

**国外部署后应该自动解决**，因为：
- Windows 服务器可以直接访问外网
- Xray 配置为 `freedom` 模式（直接出站）

## 六、安全建议

1. **修改 UUID**：生产环境使用更强的 UUID
   ```powershell
   [guid]::NewGuid().ToString()
   ```

2. **配置防火墙**：仅允许必要的 IP 访问
   ```powershell
   # 查看当前规则
   Get-NetFirewallRule -DisplayName "Xray VPN Port"
   ```

3. **定期更新**：
   ```powershell
   cd E:\docs\xray-vpn
   docker compose pull
   docker compose up -d
   ```

4. **监控日志**：
   ```powershell
   docker compose logs -f
   ```
